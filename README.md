

# 🕋 Steps of Muhammad ﷺ — Digital Litany Repository

Welcome to the technical architecture and content management guide for **Steps of Muhammad ﷺ**. This repository functions as a lightweight, lightning-fast, zero-build web application hosted entirely via GitHub Pages. 

This guide is designed to make it incredibly easy to add new interactive Du'as, modify the design, and maintain clean folder organization—even if you are completely new to web development.

## 🧩 Understanding the Core Files

To manage this project seamlessly, you only need to understand three core concepts: **The Paint**, **The Bricks**, and **The Map**.

### 1. `dua-core.css` (The Paint)

* **What it does:** This single file holds all the colors, fonts, margins, animations, and the layout engine for the interactive zoom docks.
* **The Superpower:** Instead of copying 150+ lines of design code into every single Du'a page, every page simply links to this file. If you want to change the theme color from emerald green to gold, you change it **once** in this file, and every Du'a page across your entire website updates instantly.

### 2. `dua-xyz.html` (The Bricks)

* **What it does:** These are individual Du'a pages generated by AI (Claude). They contain **zero design code**. They only contain raw text, Arabic script, English translations, and the specific per-word breakdown items.
* **How they get styled:** At the top of every slim Du'a file, there is a line that says: `<link rel="stylesheet" href="../dua-core.css">`. The user's browser reads this line, grabs the master CSS file from the root, and builds the beautiful interface automatically.

### 3. `index.html` (The Map)

* **What it does:** This is your home screen dashboard. It contains the logic for the dynamic category cards and the automated audio player engine.
* **How it organizes:** It uses a Javascript configuration array (`ROOT_CATEGORIES`) to safely point users toward your subfolders.

## 🔄 The Day-to-Day Workflow (Adding a New Du'a)

Whenever you want to add a new interactive Du'a to your collection, follow this exact 4-step process:

### Step 1: Prepare your Tools

Make sure you have your raw materials ready:

1. The **Hadith text, source, and authenticity grade**.
2. Your local **`dua-shell-template.html`** file (the empty structural mold you feed to Claude).

### Step 2: Prompt Claude

Send a message to Claude containing the template and the raw text.

* *Note: If you are using Claude Projects, make sure your updated system instructions are saved in the project instructions!*

**Example Chat Message:**

> Attached is the `dua-shell-template.html`. Please generate a slim HTML file for this source data:
> **Arabic:** اللَّهُمَّ إِنِّي أَسْأَلُكَ...
> **Source:** Sahih Muslim 1234
> **Grade:** Sahih

### Step 3: Save and Upload to GitHub

1. Copy Claude’s HTML output.
2. Save it locally as a web file (e.g., `dua-istikhara.html`).
3. Upload this file directly into the correct category subfolder on GitHub (e.g., drop it inside `/dua-shifa/` or `/wird/`).

### Step 4: Update the Navigation Map

Because this is a lightweight website with no automated database backend, **you must manually update your index file** so users can find your new page.

* **If routing a homepage card directly to a single file:** Open your root `index.html`, find `ROOT_CATEGORIES`, and change the `path` variable to point straight to your new file:

```javascript
  path: "dua-haj/dua-safar.html",

```

* **If using a category index menu:** Open the `index.html` file inside that specific subfolder and add a new link text pointing to your uploaded file:

```html
  <a href="dua-healing.html">Du'a for Healing</a>

```

## ⚠️ Crucial Rules for Maintenance (Avoid Broken Layouts)

* **Rule #1: Mind the Dots (`../`)**
Because your individual Du'a files live inside subfolders (one level deep), they must look "up and out" to find the CSS file at the root. Ensure the top layout link in your Du'a pages is exactly:

```html
  <link rel="stylesheet" href="../dua-core.css">

```

* **Rule #2: No Spaces in Folder Names**
Web servers break when folders have empty spaces. Never name a folder `Dua Shifa/`. Instead, use clean, lowercase dashes: `dua-shifa/`.
* **Rule #3: Leave `dua-core.css` Alone**
Do not move `dua-core.css` inside subfolders. It must sit at the absolute top-level root of your repository forever.
